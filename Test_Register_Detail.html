Detailed Test Plan: User Registration (10 Scenarios)

ID Test Scenario Test Procedure (Detailed Actions) Expected Result Pre-conditions
REG_01 Successful Registration
  1. Open the web browser (Chrome/Edge).
  2. Enter the URL: localhost:8080/fresh-food-store/register.
  3. Click on the Full Name field and type "Nguyen Van A".
  4. Click on the Email field and type a new email "testuser@gmail.com".
  5. Click on the Phone field and type "0912345678".
  6. Click on the Password field and type "Abc123456".
  7. Click the Register button.
  8. Open a new tab and log in to the test email account.
  9. Open the latest email from "Fresh Food Store" and copy the 6-digit OTP.
  10. Return to the website and paste the OTP into the verification box.
  11. Click the Verify button.
 - System displays a popup or message: "Xác thực thành công!"
- Page automatically redirects to /login.
- New record for "testuser@gmail.com" appears in the database. 		- Browser is working.
- Internet connection active.
- Database is running.
REG_02 Register with Existing Email
  1. Navigate to the Register page.
  2. Type an email address that is already registered in the system (e.g., "admin@gmail.com").
  3. Fill in other fields with valid dummy data.
  4. Click the Register button.
 - System stops the registration process.
- Red error message appears: "Email này đã tồn tại trong hệ thống."
- No OTP is sent to the email address. 		Target email must exist in the Accounts table.
REG_03 Empty Required Fields
  1. Navigate to the Register page.
  2. Leave the Full Name field blank.
  3. Fill in Email, Phone, and Password.
  4. Click the Register button.
 - Browser shows a tooltip: "Please fill out this field."
- Or system displays error: "Vui lòng điền đầy đủ thông tin!"
- Page does not refresh or submit. 		HTML5 validation active.
REG_04 Invalid Email Format
  1. Navigate to the Register page.
  2. Type "invalid-email-format" (no @ symbol) in the Email box.
  3. Fill in other fields correctly.
  4. Click the Register button.
 - System displays validation error: "Email không đúng định dạng!"
- User stays on the registration page. 		ValidationUtils logic active.
REG_05 Invalid Phone Format
  1. Navigate to the Register page.
  2. Type '1234567890' (Starts with 1 instead of 0) in the Phone box.
  3. Fill in other fields correctly.
  4. Click the Register button.
 - System displays error: "Số điện thoại phải bắt đầu bằng số 0 và bao gồm đúng 10 chữ số!"
- Data is not processed.
REG_06 Weak Password Violation
  1. Navigate to the Register page.
  2. Type "simple" (all lowercase, no numbers) in the Password box.
  3. Fill in other fields correctly.
  4. Click the Register button.
 - System displays error: "Mật khẩu phải từ 8 ký tự, bao gồm ít nhất 1 chữ hoa, 1 chữ thường và 1 số!" ValidationUtils rules applied.
REG_07 Resend OTP Cooldown Check
  1. Complete the registration form and click Register.
  2. Wait for the OTP page to load.
  3. Immediately click the Gửi lại mã (Resend) button within 5 seconds.
 - System blocks the request.
- Message appears: "Vui lòng đợi [X] giây nữa để yêu cầu mã mới." 		ResendOtpController cooldown: 60s.
REG_08 OTP Code Expiration
  1. Perform registration to receive the OTP code.
  2. Stay on the OTP verification page for exactly 130 seconds.
  3. Type the correct OTP code and click Verify.
 - System displays error: "Mã xác thực đã hết hạn (2 phút). Vui lòng đăng ký lại!"
- Redirects user back to the Register page. 		Session expiry: 120,000ms.
REG_09 Maximum OTP Failures
  1. Enter the registration flow to reach the OTP page.
  2. Type "000000" and click Verify.
  3. Repeat Step 2 four more times (Total 5 times).
 - On the 5th attempt, system displays: "Bạn đã nhập sai mã OTP quá 5 lần. Vui lòng đăng ký lại!"
- Verification session is destroyed. 		Max retry logic: 5.
REG_10 Sanitization of Name Field (XSS)
  1. In the Full Name field, type: <script>alert('Hacked')</script>.
  2. Complete registration and verify OTP.
  3. Admin logs in and views the user list.
 - No alert box appears in the Admin's browser.
- The script string is displayed as plain text or cleaned. 		Admin account exists to view list.
