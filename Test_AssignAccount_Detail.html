Detailed Test Plan: Admin User Management (10 Scenarios)

ID Test Scenario Test Procedure (Detailed Actions) Expected Result Pre-conditions
ADM_01 Successful Account Assignment
  1. Log in to the system with an Admin account.
  2. Click on the Admin Dashboard.
  3. Navigate to the Assign Account menu item.
  4. Select "Staff" from the Role dropdown list.
  5. Type "Staff Member A" in the Full Name box.
  6. Type a unique email "staff_a@ffs.vn" in the Email box.
  7. Click the Assign button.
 - System displays message: "Assign success".
- Account is created with password "FreshFood123".
- Notification email is sent to the target email. 		Current user must have Role ID = 1.
ADM_02 Assignment with Missing Data
  1. Navigate to the Assign Account page.
  2. Leave the Email field completely empty.
  3. Fill in Name and Role.
  4. Click the Assign button.
 - System blocks submission.
- Error message appears: "Vui lòng nhập đầy đủ Họ tên, Email và chọn Vai trò!" 		Handled by AdminController.
ADM_03 Assignment with Invalid Data Format
  1. Navigate to the Assign Account page.
  2. Type '1234567890' (Starts with 1 instead of 0) in the Phone field.
  3. Type an invalid email format.
  4. Click the Assign button.
 - System displays error: "Email không đúng định dạng!" hoặc "Số điện thoại phải bắt đầu bằng số 0 và bao gồm đúng 10 chữ số!"
- No database entry is created.
ADM_04 Ban an Active User
  1. Navigate to the User Management list page.
  2. Locate a user with Status "Active".
  3. Click the Ban (or Status Toggle) button.
  4. Click "OK" on the confirmation prompt (if any).
 - User status column updates to "Banned".
- The user's row color may change to grey/red. 		Target user must not be an Admin.
ADM_05 Admin Self-Ban Restriction
  1. Identify your own Admin account in the User List.
  2. Click the Ban button next to your name.
 - System rejects the action.
- Error message appears: "Không thể khóa tài khoản Quản trị viên!" 		Code check: target.getRoleId() == 1.
ADM_06 Modify User Role and Verify Email
  1. Click the Edit button for a Staff member.
  2. Change the Role from 'Staff' to 'Shipper'.
  3. Click the Save button.
 - Database updates the role ID for that user.
- User receives an automated email about the role change. 		Handled in EditUserController.
ADM_07 User List Pagination
  1. Navigate to the User List page.
  2. Scroll to the bottom navigation bar.
  3. Click on the number 2 (or 'Next') button.
 - Table loads the next 10 user records.
- The browser URL updates to /admin/users?index=2. 		Total users in DB must exceed 10.
ADM_08 Search by Partial Name
  1. Locate the Search bar at the top of the user list.
  2. Type "Ngu" (partial string for Nguyen).
  3. Press Enter or click the Search icon.
 - Table only displays users whose names contain "Ngu".
- Counter updates: "Found [X] records". 		Dynamic SQL in AccountDAO.
ADM_09 Role Filtering Logic
  1. Open the Filter by Role dropdown.
  2. Select 'Shipper'.
  3. Click the Apply Filter button.
 - Table filters out all users except those with Role ID = 4 (Shipper).
ADM_10 Unauthorized Admin Access Block
  1. Logout from the Admin account.
  2. Log in as a regular Customer.
  3. Manually type the URL: localhost:8080/fresh-food-store/admin/assign in the browser.
 - System denies access.
- User is redirected to Login or an "Access Denied" page. 		Access control handled by LoginFilter.
